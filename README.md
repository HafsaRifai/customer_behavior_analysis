# 🛍️ Customer Behavior Analysis

A complete end-to-end data analytics project analyzing retail customer shopping trends using Python, PostgreSQL, and Power BI.

---

## 📌 Overview

This project follows an industry-standard data analytics workflow — from raw data ingestion and cleaning, through SQL-based analysis, to an interactive Power BI dashboard. The goal was to uncover actionable insights about customer spending behavior, product performance, and subscription patterns from a retail dataset of 3,900 customers.

---

## 📊 Dataset

- **Source:** Customer Shopping Behavior Dataset
- **Size:** 3,900 records, 18 columns
- **Domain:** Retail — customer demographics, purchase history, product categories, shipping, and subscription data

## 🧹 Data Cleaning & Feature Engineering

Performed in `customer_bahavior_analysis.ipynb`:

- **Missing values:** 37 null values in `Review Rating` — imputed using **category-wise median** to preserve distribution accuracy
- **Column standardisation:** Renamed all columns to lowercase with underscores for PostgreSQL compatibility
- **Duplicate column removal:** Discovered `discount_applied` and `promo_code_used` were 100% identical — dropped the redundant column
- **Feature: Age Group** — created `age_group` using quartile-based binning (Youth Adults / Adults / Middle-aged / Senior)
- **Feature: Purchase Frequency Days** — mapped text-based frequency labels (e.g. "Fortnightly", "Weekly") to numeric day values for quantitative analysis
- **Data export:** Cleaned dataset loaded into PostgreSQL using SQLAlchemy for SQL analysis

## 🗄️ SQL Analysis

The `sql query analysis.sql` file contains 10 business-driven queries on the cleaned PostgreSQL database. Key techniques include aggregations, subqueries, CTEs, CASE statements, and window functions.

**Business Questions Answered:**

**Revenue & Spending Patterns**
- Compared total revenue generated by male vs. female customers using GROUP BY aggregation
- Analysed whether subscribed customers spend more by comparing average spend and total revenue between subscribers and non-subscribers
- Broke down revenue contribution by age group to identify the highest-value demographics

**Product Performance**
- Identified the top 5 highest-rated products using AVG with ROUND and ORDER BY
- Found the 5 products with the highest discount usage rates using conditional aggregation (CASE + SUM)
- Retrieved the top 3 most purchased products within each category using a CTE with ROW_NUMBER() window function partitioned by category

**Customer Behaviour**
- Identified high-value discount users — customers who applied a discount yet still spent above the average purchase amount, using a correlated subquery
- Segmented customers into New, Returning, and Loyal groups based on purchase history using a CTE with CASE logic
- Investigated whether repeat buyers (5+ previous purchases) are more likely to subscribe
- Compared average purchase amounts between Standard and Express shipping types

## 💡 Key Insights

- **Youth Adults generate the highest revenue and transaction volume** across all age groups, making them the most valuable customer segment
- **Clothing is the top-performing category** by both revenue and number of sales
- **73% of customers are non-subscribers**, indicating a significant opportunity to grow the subscription base
- Customers who applied discounts and still spent above average represent a high-value segment worth targeting with loyalty programs
- Discount and promo code usage were found to be identical across all records — suggesting a system-level coupling between the two that may be worth reviewing operationally

## 🚀 How to Run

### 1. Clone the Repository
```
git clone https://github.com/HafsaRifai/customer_behavior_analysis.git
cd customer_behavior_analysis
```

### 2. Install Dependencies
```
pip install pandas sqlalchemy psycopg2-binary jupyter
```

### 3. Run the Jupyter Notebook
- Open `customer_bahavior_analysis.ipynb`
- Update the PostgreSQL connection string with your local credentials
- Run all cells to clean the data and load it into PostgreSQL

### 4. Run SQL Queries
- Open PostgreSQL (pgAdmin or psql)
- Connect to the `customer_behavior` database
- Run queries from `sql query analysis.sql`

### 5. Open Power BI Dashboard
- Open `customer_behavior_dashboard.pbix` in Power BI Desktop
- Refresh the data connection if required
